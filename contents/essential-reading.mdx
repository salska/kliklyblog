**Kickstart your transition from vibe coding to agentic engineering.** 

If you're still treating AI like autocomplete, you're already behind.

Agentic engineering isn’t about writing code faster—it’s about designing systems where code gets written, verified, and maintained with minimal human intervention. Most content online still misses that.

This list doesn’t.

---

## How to Use Claude Code Effectively

[Read the article](https://spiess.dev/blog/how-i-use-claude-code) by Philipp Spiess • June 2025 • 14 min

Still one of the clearest “day 1 → day 30” transitions into agent workflows—but read it differently in 2026:

- **Context management is the real skill**: `/clear` isn’t a tip, it’s survival  
- **Prompting is specification**: Treat prompts like contracts, not suggestions  
- **Iteration beats intelligence**: Good loops outperform clever prompts  
- **Git is your memory layer**: The model forgets—your repo can’t  

**What’s changed:**  
This isn’t about “using Claude well” anymore—it’s about **controlling a non-deterministic system**.

> The gap between beginners and experts is no longer coding ability—it's how they manage context and failure.

## Agentic Coding: The Future of Software Development with Agents

[Watch the video](https://www.youtube.com/watch?v=nfOVgz_omlU) by Armin Ronacher • June 2025 • 37 min

A foundational mental model for agentic systems:

- **Agents amplify system quality**: Bad logs, vague errors → catastrophic loops  
- **Terminal workflows still dominate**: Composability > UX polish  
- **Simple stacks win**: The more “clever” your codebase, the worse agents perform  
- **Restarting is a feature**: Long-running context is a liability  

**What’s changed:**  
We now know this isn’t optional—**teams that ignore this hit a hard ceiling fast**.

> Agent performance is mostly a reflection of how debuggable your system is.

## I can't sleep gud anymore — A Practical Guide to Agentic Computering

[Watch the video](https://vimeo.com/1098025052) by Mario Zechner • June 2025 • 85 min

This is still the most *honest* deep dive into real usage:

- **Claude writes bad code—fast**: Speed is useless without constraints  
- **External state > context stuffing**: Files, schemas, and logs beat memory  
- **Prompts = programs**: Deterministic workflows are the endgame  
- **Tooling > model choice**: The system around the model matters more  

**What’s changed:**  
This is basically the blueprint for **modern agent pipelines** now.

> The winning pattern isn’t “ask better”—it’s “build systems the model can’t mess up.”

## Claude Code Best Practices from Anthropic

[Read the article](https://www.anthropic.com/engineering/claude-code-best-practices) by Anthropic Engineering • April 2025 • 15 min

Still relevant—but now clearly **baseline, not advanced**:

- **Explore → Plan → Code → Commit** is just standard operating procedure  
- **TDD works because it constrains the model**  
- **Parallel agents are powerful—but fragile**  
- **Subagents introduce coordination problems quickly**  

**What’s changed:**  
Most teams fail not by ignoring this—but by **stopping here**.

> Best practices help you start. They don’t help you scale.

[Read the article](https://mariozechner.at/posts/2025-08-15-mcp-vs-cli/) by Mario Zechner • August 2025 • 12 min

Cuts through one of the biggest ongoing wastes of time:

- **Most MCP tooling is unnecessary abstraction**  
- **Context pollution is a silent killer**  
- **CLI tools are still the most reliable interface**  
- **Simplicity scales better than clever integrations**  

**What’s changed:**  
People learned the hard way: **more tools ≠ better agents**.

Still the best framing for what’s actually happening:

- **LLMs behave like unstable operating systems**

## What’s Missing (and Matters Now)

If you only read the above, you’re still behind current practice. The real gaps:

- **Evaluation systems** (tests for agents, not just code)  
- **Deterministic pipelines** (reducing variance)  
- **Observability for agents** (logs, traces, replay)  
- **Failure design** (what happens when—not if—it breaks)  

> That’s where the field actually is so far in 2026.
